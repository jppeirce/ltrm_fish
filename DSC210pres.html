Spatial and Temporal Changes in Fish Assemblages of the Upper Mississippi River

This project is a collaboration with federal scientists at the US Geological Survey (USGS) Upper Midwest Environmental Sciences Center (UMESC).

For over 30 years, the USGS UMESC has been monitoring the fish assemblages of the Upper Mississippi River as part of the Long Term Resource Monitoring (LTRM) program. The LTRM program collects data on fish populations, habitat conditions, and other environmental factors to understand the spatial and temporal changes in fish assemblages. This information is crucial for informing conservation and management efforts to protect and sustain the fish populations in the Upper Mississippi River.

Do fish of different species hang out together?

A fish assemblage is a group of fish species that are found together in a particular habitat or area. Fish assemblages can be influenced by various factors such as water quality, habitat structure, and food availability.

Distance

How do we measure similarity between samples? We can use Jaccard distance, which is a common metric for binary data (presence/absence). The Jaccard distance between two samples A and B is defined as: \[d_J(A, B) = 1 - \frac{|A \cap B|}{|A \cup B|}\] Where \(|A \cap B|\) is the number of species present in both samples, and \(|A \cup B|\) is the number of species present in either sample.

How do we decide how many clusters to cut the dendrogram into?

There is no single “correct” way to determine the optimal number of clusters, but there are several complementary approaches we can use to make an informed decision.

Method 1 — Largest Gap

  • Idea: Find the biggest jump between consecutive merge heights and cut in the middle.
  • Pros: Simple, parameter-free, and purely visual — it formalizes the “look for big gaps” heuristic.
  • Cons: Only considers one gap. If the dendrogram has several similarly-sized gaps it can be unstable. It also ignores cluster quality entirely — a large gap doesn’t guarantee internally cohesive groups.

Method 2 — Silhouette Score

  • Idea: For each threshold, assign clusters and compute the mean silhouette coefficient, which balances within-cluster tightness against between-cluster separation. Range is [−1, 1]; higher is better.
  • Pros: Directly measures cluster quality using the same Jaccard distance matrix. Works well with arbitrary cluster shapes and is agnostic to the clustering method.
  • Cons: Computationally expensive on large datasets (\(O(n^2)\) per evaluation). Can favor coarser (fewer) clusters when many samples are dissimilar. For binary ecological data where species composition changes gradually across the landscape, silhouette values may stay low everywhere.

Method 3 — Elbow Method (Within-Cluster Sum of Squares)

  • Idea:
  1. For each candidate number of clusters k (2 through n−1), cut the dendrogram to get k groups.
  2. Compute the within-cluster sum of pairwise Jaccard distances (WCSS) — the total “spread” inside all clusters.
  3. Plot k vs. WCSS. As k increases, WCSS decreases monotonically (more clusters → each is tighter).
  4. The elbow is the k where the curve bends — further splits yield only marginal reduction in WCSS.
  • Pros: Intuitive…directly answers “at what point do we stop gaining much by splitting further?”
  • Cons: The elbow can be ambiguous — ecological data often produces a smooth, gradual curve with no sharp bend, making the “elbow” subjective. Biased toward many clusters — WCSS always decreases with more clusters, so the method can’t tell you when clusters become meaningless, only when they become marginally less helpful.
Show this code 
# =========================================================================
# Method: Elbow Method (Within-Cluster Sum of Squares / Inertia)
# =========================================================================
# For each possible number of clusters k (from 2 to n-1), cut the
# dendrogram and compute the total within-cluster sum of squared Jaccard
# distances. Plot k vs. WCSS and look for the "elbow" — the point where
# the curve bends and additional clusters stop providing much improvement.

subset_binary = fish_df_binary.iloc[0:20]
dist_sub = pdist(subset_binary, metric='jaccard')
Z_sub = linkage(dist_sub, method='average')
dist_square_sub = squareform(dist_sub)

max_k = len(subset_binary) - 1  # max clusters = n-1
k_range = range(2, max_k + 1)
wcss = []

for k in k_range:
    labels = fcluster(Z_sub, t=k, criterion='maxclust')
    total_wcss = 0
    for c in np.unique(labels):
        members = np.where(labels == c)[0]
        if len(members) > 1:
            # Sum of pairwise Jaccard distances within this cluster
            cluster_dists = dist_square_sub[np.ix_(members, members)]
            # Use upper triangle to avoid double-counting
            total_wcss += np.sum(np.triu(cluster_dists, k=1))
    wcss.append(total_wcss)

wcss = np.array(wcss)
k_array = np.array(list(k_range))

# ── Automated elbow detection via maximum curvature ──
# Draw a line from the first point to the last point; the elbow is the
# k whose perpendicular distance to that line is greatest.
p1 = np.array([k_array[0], wcss[0]])
p2 = np.array([k_array[-1], wcss[-1]])
line_vec = p2 - p1
line_len = np.linalg.norm(line_vec)
line_unit = line_vec / line_len

distances_to_line = []
for i in range(len(k_array)):
    pt = np.array([k_array[i], wcss[i]])
    proj = np.dot(pt - p1, line_unit)
    closest = p1 + proj * line_unit
    distances_to_line.append(np.linalg.norm(pt - closest))

elbow_idx = np.argmax(distances_to_line)
elbow_k = k_array[elbow_idx]

print("── Elbow Method (Within-Cluster Sum of Jaccard Distances) ──")
print(f"  Suggested number of clusters (elbow) = {elbow_k}")

# Recover the merge-height threshold that gives elbow_k clusters
labels_elbow = fcluster(Z_sub, t=elbow_k, criterion='maxclust')
# The threshold is the merge height just before the (n - elbow_k)th merge
threshold_elbow = Z_sub[-(elbow_k), 2] if elbow_k <= len(Z_sub) else Z_sub[0, 2]
print(f"  Corresponding distance threshold ≈ {threshold_elbow:.4f}")
print(f"  Cluster sizes = {np.bincount(labels_elbow)[1:]}")

# ── Plot ──
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

ax = axes[0]
ax.plot(k_array, wcss, 'o-', ms=5)
ax.axvline(elbow_k, color='red', ls='--', label=f'Elbow at k = {elbow_k}')
ax.set_xlabel('Number of clusters (k)')
ax.set_ylabel('Within-cluster sum of Jaccard distances')
ax.set_title('Elbow Method')
ax.legend()

ax = axes[1]
dendrogram(Z_sub, ax=ax, color_threshold=threshold_elbow)
ax.axhline(threshold_elbow, color='red', ls='--')
ax.set_title(f'Dendrogram cut at k = {elbow_k} (d ≈ {threshold_elbow:.3f})')
ax.set_xlabel('Sample index')
ax.set_ylabel('Distance')

plt.tight_layout()
plt.show()

For example, if we want to identify broad assemblage types that are easy to interpret and communicate, we might prefer a smaller number of clusters (e.g. 4-8) that capture major patterns in the data. On the other hand, if we want to explore finer-scale variation in assemblages or identify rare or specialized groups, we might prefer a larger number of clusters (e.g. 10-20) that capture more subtle differences between samples.
